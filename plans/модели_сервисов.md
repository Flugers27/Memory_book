# Модели, используемые в сервисах

## Сервис Auth (авторизация)

Модели определены в `services/Auth/models.py`:

| Модель | Таблица | Описание |
|--------|---------|----------|
| `UserRole` | `user_roles` | Роли пользователей с правами |
| `User` | `users` | Пользователи системы |
| `RefreshToken` | `sys.refresh_tokens` | Токены обновления (в схеме sys) |
| `EmailVerificationToken` | `email_verification_tokens` | Токены верификации email |

**Поля UserRole:**
- `id_role` (UUID, PK)
- `role_name` (String, unique)
- `role_description` (Text)
- `permissions` (JSON)
- `is_system` (Boolean)
- `created_at`, `updated_at` (DateTime)

**Поля User:**
- `id_user` (UUID, PK)
- `email` (String, unique)
- `username` (String, unique)
- `full_name` (String)
- `password_hash` (String)
- `is_active` (Boolean)
- `is_verified` (Boolean)
- `role_id` (UUID, FK → user_roles.id_role)
- `last_login_at` (DateTime)
- `created_at`, `updated_at` (DateTime)

**Поля RefreshToken:**
- `id_user_token` (UUID, PK)
- `user_id` (UUID)
- `token_hash` (String)
- `device_info` (Text)
- `ip_address` (String)
- `expires_at` (DateTime)
- `created_at` (DateTime)

**Поля EmailVerificationToken:**
- `id_token` (UUID, PK)
- `user_id` (UUID)
- `token` (String, unique)
- `expires_at` (DateTime)
- `created_at` (DateTime)

## Сервис Memory (страницы памяти)

Модели определены в `services/Memory/models.py` и `database/models/memory.py` (дублируются):

| Модель | Таблица | Описание |
|--------|---------|----------|
| `AgentBD` | `agents` | Агенты (люди/персоны) |
| `PageBD` | `pages` | Страницы памяти |

**Поля AgentBD:**
- `id_agent` (String(36), PK)
- `full_name` (String)
- `gender` (String(1))
- `birth_date` (Date)
- `death_date` (Date)
- `place_of_birth` (String)
- `place_of_death` (String)
- `avatar_url` (Text)
- `created_at`, `updated_at` (DateTime)
- `user_id` (String(36))
- `is_human` (Boolean)

**Поля PageBD:**
- `id_page` (String(36), PK)
- `epitaph` (Text)
- `is_public` (Boolean)
- `is_draft` (Boolean)
- `agent_id` (String(36), FK → agents.id_agent)
- `created_at`, `updated_at` (DateTime)
- `user_id` (String(36))
- `biography` (JSON)

**Связи:**
- `AgentBD.pages` → список `PageBD`
- `PageBD.agent` → `AgentBD`
- `PageBD.access` → `PageAccessControl` (из сервиса Acces_Memory)

## Замечания

1. **Дублирование моделей**: Некоторые модели присутствуют и в `services/*/models.py`, и в `database/models/`. Например, `AgentBD` и `PageBD` есть в обоих местах. Это может привести к конфликтам импорта.

2. **Ссылки на несуществующие модели**: В `PageAccessControl` есть ForeignKey на `memory_page.id_page` и `auth_user.id_user`, но соответствующих моделей `MemoryPage` и `AuthUser` в коде не найдено. Возможно, это ссылки на таблицы из других сервисов, которые должны быть синхронизированы.

3. **Использование UUID vs String**: В одних моделях первичный ключ типа `UUID` (PostgreSQL UUID), в других — `String(36)`. Нужно привести к единообразию.

4. **Схемы БД**: `RefreshToken` использует схему `sys`, остальные — `public`.

5. **Отношения**: Не все отношения явно прописаны (например, между `PageBD` и `MediaBD`).

Рекомендуется провести рефакторинг моделей для устранения дублирования и обеспечения согласованности.
