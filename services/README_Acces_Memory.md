# Access Management Service

## Назначение

Сервис Access Management отвечает за управление доступом к страницам памяти пользователей. Он предоставляет REST API для:

- проверки наличия доступа у пользователя к конкретной странице;
- получения списка страниц, к которым пользователь имеет доступ;
- получения списка доступов, предоставленных пользователем другим;
- предоставления и отзыва доступа к страницам;
- обновления параметров доступа (права на просмотр/редактирование, срок действия);
- полного удаления записей доступа.

## Основные модули

- `main.py` — точка входа FastAPI-приложения.
- `config.py` — конфигурация сервиса (порт, JWT-ключи, CORS, пагинация).
- `crud.py` — операции доступа к данным (SQL-запросы, проверки прав).
- `dependencies.py` — валидация JWT и получение `user_id`.
- `schemas.py` — Pydantic-схемы запросов и ответов.
- `routers/access.py` — основные маршруты управления доступом.
- `routers/health.py` — health-check.

## Роуты

### Доступ к страницам

- `GET /access/my` — список страниц, к которым текущий пользователь имеет доступ (с пагинацией).
- `GET /access/granted` — список доступов, предоставленных текущим пользователем другим (с пагинацией).
- `GET /access/page/{page_id}/check` — проверка доступа текущего пользователя к конкретной странице.
- `GET /access/page/{page_id}/full` — полная информация о странице, агенте и доступе (только при наличии доступа).

### Управление доступом

- `POST /access/grant` — предоставить доступ к странице другому пользователю.
- `PUT /access/{access_id}` — обновить параметры доступа (права, срок действия).
- `DELETE /access/{access_id}` — отозвать доступ (мягкое удаление, деактивация).
- `DELETE /access/hard/{access_id}` — полностью удалить запись доступа из БД.

### Health

- `GET /health` — проверка состояния сервиса.

## Авторизация

Для защищённых маршрутов сервис использует JWT. В `dependencies.py` токен декодируется локально с использованием `SECRET_KEY` из конфига, и из поля `sub` извлекается `user_id`. Используется схема `HTTPBearer` (поле "Bearer token" в Swagger UI).

## Запуск

Из корня проекта:

```bash
cd services/Acces_Memory
python run.py
```

Или через FastAPI/uvicorn:

```bash
python -m uvicorn services.Acces_Memory.main:app --reload
```

## Примечания

- Все маршруты, кроме `/health`, требуют авторизации (JWT-токен в заголовке `Authorization: Bearer <token>`).
- Владелец страницы имеет полный доступ (просмотр и редактирование) без необходимости создавать запись в `page_access_control`.
- Публичные страницы (не черновики) доступны для просмотра всем авторизованным пользователям без дополнительной записи доступа.
- Для локального развёртывания важно, чтобы переменные окружения (`SECRET_KEY`, `DATABASE_URL`, `ACCESS_PORT`) были правильно заданы.
- Сервис работает на порту **8003** по умолчанию.
