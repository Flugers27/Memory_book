# Family Tree Service

## Назначение

Сервис Family Tree отвечает за создание и управление генеалогическими древами пользователей. Он предоставляет REST API для:

- создания, просмотра, обновления и удаления семейных древ;
- добавления и удаления агентов (членов семьи) в древе;
- управления связями между агентами (родственные отношения);
- получения полного древа со всеми агентами и связями;
- публикации и чтения публичных семейных древ.

## Основные модули

- `main.py` — точка входа FastAPI-приложения.
- `config.py` — конфигурация сервиса (порт, JWT-ключи, CORS, пагинация).
- `crud.py` — операции доступа к данным (SQL-запросы для работы с деревьями, агентами и связями).
- `dependencies.py` — валидация JWT и получение `user_id`.
- `schemas.py` — Pydantic-схемы запросов и ответов.
- `routers/family.py` — основные маршруты для работы с семейными древами.
- `routers/health.py` — health-check.

## Модели данных

### FamilyTree (`family_tree`)

| Поле | Тип | Описание |
|------|-----|----------|
| `id_family_tree` | UUID (PK) | Идентификатор древа |
| `name_family_tree` | VARCHAR(255) | Название древа |
| `is_public` | BOOLEAN | Публичный доступ |
| `is_draft` | BOOLEAN | Черновик |
| `user_id` | UUID (FK → users) | Владелец древа |
| `created_at` | TIMESTAMPTZ | Дата создания |
| `updated_at` | TIMESTAMPTZ | Дата обновления |

### FamilyTreeAgent (`family_tree_agents`)

| Поле | Тип | Описание |
|------|-----|----------|
| `id_tree_agent` | UUID (PK) | Идентификатор записи |
| `family_tree_id` | UUID (FK → family_tree) | Древо |
| `agent_id` | UUID (FK → agents) | Агент (член семьи) |

### RelationshipAgent (`relationships_agents`)

| Поле | Тип | Описание |
|------|-----|----------|
| `id_relationships` | UUID (PK) | Идентификатор связи |
| `type_relative` | VARCHAR(50) | Тип родства (например: "parent", "child", "spouse", "sibling") |
| `is_blood_relative` | BOOLEAN | Кровное родство |
| `agent_from` | UUID (FK → agents) | Агент-источник |
| `agent_to` | UUID (FK → agents) | Агент-цель |
| `family_tree_id` | UUID (FK → family_tree) | Древо |
| `user_id` | UUID (FK → users) | Пользователь, создавший связь |
| `created_at` | TIMESTAMPTZ | Дата создания |
| `updated_at` | TIMESTAMPTZ | Дата обновления |

## Роуты

Все маршруты находятся за префиксом `/family`, если не указано иное.

### Управление древами

| Метод | Эндпоинт | Описание | Авторизация |
|-------|----------|----------|-------------|
| `POST` | `/family/tree` | Создать новое генеалогическое древо | Требуется |
| `GET` | `/family/tree/my` | Список древ текущего пользователя (с пагинацией) | Требуется |
| `GET` | `/family/tree/{tree_id}` | Просмотр конкретного древа пользователя (со всеми агентами и связями) | Требуется |
| `PUT` | `/family/tree/{tree_id}` | Обновить название и настройки древа (is_public, is_draft) | Требуется |
| `DELETE` | `/family/tree/{tree_id}` | Удалить древо (каскадно удаляет связи и привязки агентов) | Требуется |

### Публичные древа

| Метод | Эндпоинт | Описание | Авторизация |
|-------|----------|----------|-------------|
| `GET` | `/family/tree/public` | Список публичных древ (для всех пользователей) | Не требуется |
| `GET` | `/family/tree/public/{tree_id}` | Просмотр конкретного публичного древа (для всех пользователей) | Не требуется |

### Агенты в древе

| Метод | Эндпоинт | Описание | Авторизация |
|-------|----------|----------|-------------|
| `POST` | `/family/tree/{tree_id}/agent` | Добавить агента (члена семьи) в древо | Требуется |
| `DELETE` | `/family/tree/{tree_id}/agent/{agent_id}` | Удалить агента из древа | Требуется |

### Связи между агентами

| Метод | Эндпоинт | Описание | Авторизация |
|-------|----------|----------|-------------|
| `GET` | `/family/tree/{tree_id}/relationship` | Список всех родственных связей в древе | Требуется |
| `POST` | `/family/tree/{tree_id}/relationship` | Создать родственную связь между двумя агентами | Требуется |
| `PUT` | `/family/tree/{tree_id}/relationship/{rel_id}` | Обновить тип родства и параметры связи | Требуется |
| `DELETE` | `/family/tree/{tree_id}/relationship/{rel_id}` | Удалить родственную связь | Требуется |

### Health

| Метод | Эндпоинт | Описание | Авторизация |
|-------|----------|----------|-------------|
| `GET` | `/health` | Проверка состояния сервиса | Не требуется |

## Авторизация

Для защищённых маршрутов сервис использует JWT. В `dependencies.py` токен декодируется локально с использованием `SECRET_KEY` из конфига, и из поля `sub` извлекается `user_id`. Используется схема `HTTPBearer` (поле "Bearer token" в Swagger UI). Владелец древа имеет полный доступ к управлению. Публичные древа доступны для чтения всем пользователям (без авторизации).

## Запуск

Из корня проекта:

```bash
cd services/Family_Tree
python run.py
```

Или через FastAPI/uvicorn:

```bash
python -m uvicorn services.Family_Tree.main:app --reload
```

## Примечания

- Все маршруты, кроме `/health` и публичных (`/family/tree/public`, `/family/tree/public/{tree_id}`), требуют авторизации (JWT-токен в заголовке `Authorization: Bearer <token>`).
- При удалении древа все связанные записи (агенты в древе и связи) удаляются каскадно.
- Типы родства (`type_relative`) задаются строкой: `"parent"`, `"child"`, `"spouse"`, `"sibling"`, `"grandparent"`, `"grandchild"` и т.д.
- В схеме данных для полей `name_family_tree` и `type_relative` добавлены Pydantic-валидаторы `strip()`, которые автоматически удаляют пробелы по краям. Это необходимо для совместимости с БД, где тип `CHAR(n)` дополняет строку пробелами до фиксированной длины.
- Агенты для древа берутся из общей таблицы `agents` (сервис Memory), что позволяет переиспользовать существующие записи о людях.
- Для локального развёртывания важно, чтобы переменные окружения (`SECRET_KEY`, `DATABASE_URL`, `FAMILY_PORT`) были правильно заданы.
- Сервис работает на порту **8005** по умолчанию.
