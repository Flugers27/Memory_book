# Memory Service

## Назначение

Сервис Memory отвечает за управление памятью/страницами пользователя, агентами и публичными карточками памяти. Он предоставляет REST API для:

- списка и просмотра агентов памяти;
- создания, обновления и удаления страниц памяти;
- публикации и чтения публичных страниц;
- проверки состояния сервиса через health-check.

## Основные модули

- `main.py` — точка входа FastAPI-приложения.
- `crud.py` — операции доступа к данным.
- `dependencies.py` — валидация JWT и получение `user_id`.
- `schemas_new.py` — Pydantic-схемы запросов и ответов (основной файл схем).
- `schemas.py` — устаревший файл схем (не используется).
- `routers/agents.py` — работа с агентами памяти.
- `routers/pages.py` — работа с личными страницами памяти.
- `routers/memory_pages.py` — работа с публичными и пользовательскими страницами памяти.
- `routers/health.py` — health-check.

## Модели данных

| Поле | Тип | Описание |
|------|-----|----------|

| `updated_at` | TIMESTAMPTZ | Дата обновления |

### Структура биографии (`biography`)

Поле `biography` хранится в формате JSONB. Сервис поддерживает два формата:

#### Новый формат (рекомендуемый)

```json
{
  "media_ids": ["uuid-медиафайла1", "uuid-медиафайла2"],
  "sections": [
    {
      "title": "Ранние годы",
      "level": 1,
      "order": 10,
      "content": "<p>Текст раздела</p>"
    },
    {
      "title": "Образование",
      "level": 1,
      "order": 20,
      "content": "<p>Текст раздела</p>"
    },
    {
      "title": "Школа",
      "level": 2,
      "order": 25,
      "content": "<p>Подраздел</p>"
    }
  ]
}
```

- `media_ids` — список UUID медиафайлов из Media Service
- `sections` — плоский массив секций с полями:
  - `title` — заголовок раздела
  - `level` — уровень вложенности (1 — основной, 2 — подраздел, 3+ — подподразделы)
  - `order` — порядок сортировки (секции сортируются по возрастанию)
  - `content` — HTML-содержимое

#### Старый формат (поддерживается для обратной совместимости)

```json
[
  {
    "title": "Ранние годы",
    "info": "Текст раздела",
    "titles": [
      {
        "title": "Подраздел",
        "info": "Текст подраздела",
        "titles": []
      }
    ]
  }
]
```

При загрузке из БД старый формат автоматически преобразуется в новый:
- `info` (plain text) → `content` (HTML `<p>...</p>`)
- Вложенные `titles` → секции с увеличенным `level`
- `media_ids` заполняется пустым массивом

## Роуты

### Агенты

- `GET /agent_list` — список агентов пользователя.
- `GET /agent/{agent_id}` — информация об агенте.
- `POST /agent/add` — создание агента.
- `PUT /agent/update/{agent_id}` — обновление агента.
- `DELETE /agent/del/{agent_id}` — удаление агента.

### Страницы памяти

- `GET /page_list/{agent_id}` — список страниц у агента.
- `GET /page/{page_id}` — конкретная страница.
- `POST /page/add` — создание страницы.
- `PUT /page/update/{page_id}` — обновление страницы.
- `DELETE /page/del/{page_id}` — удаление страницы.

### Публичные страницы

- `GET /public_memory_page_list` — список публичных страниц.
- `GET /public_memory_page/{agent_id}` — публичная страница конкретного агента.

### Пользовательские страницы памяти

- `GET /memory_page_list` — список страниц для текущего пользователя.
- `GET /memory_page/{agent_id}` — страницы конкретного агента пользователя.

### Health

- `GET /health` — проверка состояния сервиса.

## Авторизация

Для защищённых маршрутов сервис использует JWT. В `dependencies.py` токен декодируется локально и из `sub` извлекается `user_id`. Используется схема `HTTPBearer` (поле "Bearer token" в Swagger UI).

## Запуск

Из корня проекта:

```bash
cd services/Memory
python run.py
```

Или через FastAPI/uvicorn:

```bash
python -m uvicorn services.Memory.main:app --reload
```

## Примечания

- В текущей реализации часть маршрутов доступна только авторизованным пользователям.
- Для локального развёртывания важно, чтобы переменные окружения (`SECRET_KEY`, `DATABASE_URL`) были правильно заданы.
- Сервис работает на порту **8002** по умолчанию.
- Биография хранится в JSONB и поддерживает два формата — новый (плоские секции) и старый (рекурсивный список). Конвертация происходит автоматически при чтении из БД.
- Для создания/обновления страницы рекомендуется использовать новый формат биографии.
