## 1. Обзор проекта

**Memory_book** — веб-приложение для создания и управления персонализированными страницами памяти с микросервисной архитектурой. Проект включает:

- **Фронтенд:** React 19 с TypeScript, Next.js 15
- **Бэкенд:** FastAPI (Python) с микросервисной архитектурой
- **База данных:** PostgreSQL с SQLAlchemy 2.0
- **API Gateway:** Маршрутизация запросов на порту 8000
- **Аутентификация:** JWT токены

### 2.2. Обнаруженные проблемы

#### 2.2.1. API-маршруты (КРИТИЧЕСКАЯ)
**Проблема:** Фронтенд использует некорректный маршрут для получения публичных страниц:
- `/memory/public_memory_page/{agent_id}` → возвращает 500 (Internal Server Error)
- `/memory/memory_page/{agent_id}` → работает корректно (200 OK)

**Причина:** Ошибка в бэкенде при обработке публичных эндпоинтов или несоответствие требований к данным.

**Решение:** Исправлен API-клиент во фронтенде:
1. Изменен маршрут в `memory_front/lib/api.ts` с `/memory/public_memory_page/` на `/memory/memory_page/`
2. Добавлен маршрут `/memory/memory_page/` в список публичных URL для исключения проверки аутентификации

#### 2.2.2. Аутентификация (СРЕДНЯЯ) **Проблема:** Запросы к защищенным эндпоинтам без токена возвращают 401 (Unauthorized). **Статус:** Ожидается - нормальное поведение системы безопасности. **Рекомендация:** Убедиться, что токены обновляются корректно и интерцепторы работают правильно. #### 2.2.3.
